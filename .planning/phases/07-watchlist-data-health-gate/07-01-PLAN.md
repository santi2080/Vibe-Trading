<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP/client → tool registry | Untrusted `watchlist_path` and `now` inputs enter local filesystem-facing code. |
| Tool → local filesystem | The checker reads watchlist CSV and parquet metadata/content from local disk. |
| Tool → JSON consumer | Data values and error messages are serialized into JSON for agent/MCP clients. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering | `CheckWatchlistDataTool.execute` watchlist path | mitigate | Resolve every watchlist path through `_resolve_watchlist_path`; return JSON error on escape attempts; add pytest coverage for `../pyproject.toml`. |
| T-07-01-02 | Information Disclosure | data-health checker filesystem reads | mitigate | Do not expose arbitrary output-file writing or broad filesystem scan; use standard `_DATA_DIR = _REPO_DIR / "data"` source path per D-02. |
| T-07-01-03 | Denial of Service | malformed `now` input | mitigate | Parse only ISO timestamps with `datetime.fromisoformat`; reject invalid values with `status="error"` before checker execution. |
| T-07-01-04 | Information Disclosure | JSON serialization | mitigate | Serialize with `json.dumps(..., ensure_ascii=False)` and only return domain report fields; convert non-JSON scalar values through existing `_to_plain_json` if required. |
| T-07-01-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install tasks are planned. |
</threat_model>
