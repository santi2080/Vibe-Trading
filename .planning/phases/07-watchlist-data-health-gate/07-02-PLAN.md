<objective>
Integrate the data-health gate into both a protected watchlist analysis path and a real watchlist-based backtest entry, then prove blocking/warning behavior with deterministic tests.

Purpose: REQ-001 requires watchlist-based strategy work to inspect local data before running and to block when required local data is unhealthy. This plan makes `analyze_watchlist` a protected analysis path and upgrades one actual backtest entry so it checks the same gate before execution.
Output: `AnalyzeWatchlistTool` runs the existing checker before analysis, `scripts/backtest_trend_indicators.py` gains a watchlist-aware gate-protected execution path, and deterministic tests prove block and warn cases.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent/MCP client → `AnalyzeWatchlistTool` | Untrusted `watchlist_path`, `market_filter`, and `format` parameters enter a protected analysis path. |
| CLI user → backtest script | Untrusted `--watchlist` and time-related inputs enter a backtest entry that can trigger filesystem reads and report generation. |
| Gate → downstream analysis/backtest | Gate decision controls whether local data analysis/backtest work can proceed. |
| Analyzer/backtest result → JSON consumer | Analysis and gate details are serialized to JSON for clients or script consumers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Tampering | `AnalyzeWatchlistTool.execute` watchlist path | mitigate | Keep `_resolve_watchlist_path` as the first filesystem boundary; unsafe paths return JSON error before gate or analyzer execution. |
| T-07-02-02 | Elevation of Privilege | protected analysis/backtest paths | mitigate | Block downstream `WatchlistAnalyzer.analyze_all` and the watchlist-based backtest entry whenever `report.can_backtest` is false; add tests proving both paths stop before execution. |
| T-07-02-03 | Information Disclosure | JSON gate/error payload | mitigate | Return only `report.to_dict()` gate/rules/items and existing analysis/backtest fields; avoid raw tracebacks and arbitrary file content output. |
| T-07-02-04 | Denial of Service | oversized or stale real data dependency in tests | mitigate | Use deterministic tmp_path fixtures and fixed `now` values; do not rely on live data freshness or network fallback in tests. |
| T-07-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install tasks are planned. |
</threat_model>
