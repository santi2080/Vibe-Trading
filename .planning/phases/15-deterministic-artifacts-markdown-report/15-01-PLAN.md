    Create `agent/src/data/scan_reporting.py` with three components:

    1. **`Manifest` dataclass** — scan metadata and file inventory:
       - Fields: `scan_date: str`, `watchlist_name: str`, `version: str = "1.0.0"`, `artifacts: dict[str, str]` mapping artifact name to filename (e.g., `{"data_health": "data_health.json", "scan_results": "scan_results.json", "report": "report.md"}`), `scan_info: dict` (copied from ScanSignalReport.scan_info), `total_symbols: int`
       - `to_dict()` returning a JSON-serializable dict
       - `to_json(path: Path)` writing via `json.dumps(..., indent=2, default=str)`

    2. **`MarkdownReportRenderer` class** — renders report.md from JSON artifacts:
       - `__init__(self, scan_plan: ScanPlan, output_dir: Path, scan_date: str, signal_report: ScanSignalReport | None = None, health_report_dict: dict | None = None)`: store references; load scan_results.json and data_health.json from output_dir if not passed
       - `load_artifacts(self) -> None`: if signal_report/health_report_dict are None, load them from output_dir JSON files
       - `render(self) -> str`: produce full Markdown report string with these sections:
         - **Header**: `# Daily Scan Report — {scan_date}` with watchlist name, version
         - **Data Health**: status badge (PASS/WARNING/FAIL from data_health.json), total checks, blocking failures, warnings. If status == "FAIL": stop here with "Scan blocked — fix data issues above."
         - **Actionable Candidates** (`actionable` bucket): count, table with Symbol | Name | Market | Score | Confidence | Reason columns. Note: "Candidates are for research only. Not financial advice." (per scope guardrail)
         - **Watch List** (`watch` bucket): count, table with Symbol | Name | Market | Reason
         - **Risk / Excluded** (`risk_excluded` bucket): count, table with Symbol | Name | Market | Reason
         - **Skipped** (`skipped` bucket): count, table with Symbol | Name | Reason
         - **Failed** (`failed` bucket): count, table with Symbol | Error
         - **Artifacts**: bullet list of artifact filenames with links: `data_health.json`, `scan_results.json`, `manifest.json`, `report.md`
       - Key constraints (per success criteria #4): NO ranking, NO performance metrics, NO "best configuration", NO trading advice, NO "buy/sell/hold" language
       - Use `signal_report.buckets` and `signal_report.buckets_summary` from the loaded dict (not ad-hoc state)
       - `write(self, path: Path) -> None`: write the rendered string to path with UTF-8 encoding

    3. **`run_reporting(scan_plan: ScanPlan, output_dir: Path, *, scan_date: str, format: str, console: Console | None) -> Manifest`** — orchestrator:
       - Build `Manifest` from scan_plan and scan_date
       - Write `manifest.json` to output_dir via `manifest.to_json(output_dir / "manifest.json")`
       - Load data_health.json and scan_results.json from output_dir
       - Build `MarkdownReportRenderer` with loaded artifacts
       - Call `renderer.render()` -> `report_md`
       - Write `report.md` to output_dir
       - If format == "table" and console is not None: print a `[dim]Report written to {report_path}[/dim]` line
       - Return `manifest`

    Import path note: since this module lives in `agent/src/data/`, use relative imports from sibling modules.
