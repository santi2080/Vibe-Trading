Implement the signal scan bucket classification layer for the daily scan pipeline (SIG-01, SIG-02). Output a `rich` table in human mode and JSON in machine mode.

Purpose: Fulfills SIG-01 (strategy semantics) and SIG-02 (bucket assignment) so the user can see every symbol classified.

       - `symbol: str`, `name: str`, `market: str`
       - `bucket: Literal["actionable", "watch", "risk_excluded", "skipped", "failed"]`
       - `bucket_reason: str` — human-readable reason
       - `trading_signal: dict | None` — `TradingSignal.to_dict()` or None
       - `error: str | None` — exception message, only set when bucket == "failed"

    2. **`classify_trading_signal(signal: TradingSignal) -> tuple[str, str]`** — classify a TradingSignal into bucket + reason:
       - `INVALID` status -> `("failed", f"status=INVALID: {signal.warnings[0] if signal.warnings else 'unknown'}")`
       - `NO_SIGNAL` status -> `("skipped", "status=NO_SIGNAL: no trend signal detected")`
       - `BLOCKED` or `EXHAUSTED` readiness -> `("risk_excluded", f"readiness={readiness}")`
       - BULL or BEAR + VALID + READY:
         - confidence >= 0.45 -> `("actionable", f"{direction} trend, confidence={confidence:.2f}, score={signal_score:.1f}")`
         - confidence >= 0.25 -> `("watch", f"{direction} trend, low confidence={confidence:.2f}")`
         - confidence < 0.25 -> `("risk_excluded", f"low confidence={confidence:.2f}")`
       - `FILTERED` status -> `("risk_excluded", f"FILTERED: {signal.warnings[0] if signal.warnings else 'filtered out'}")`
       - `NEUTRAL` direction -> `("watch", "no directional trend (NEUTRAL)")`
       - default fallback -> `("watch", f"direction={signal.direction}, status={signal.status}")`
       - Constants `BULL_BEAR_THRESHOLD = 0.45` and `WATCH_CONFIDENCE_FLOOR = 0.25` at module level (per research Assumption A1)

    3. **`ScanSignalReport` dataclass** with fields:
       - `scan_info: dict` — watchlist, data_dir, output_dir, scan_date, strategy, version
       - `buckets_summary: dict[str, int]` — counts per bucket + total
       - `buckets: dict[str, list[dict]]` — keyed by bucket name, each value is a list of SymbolSignalResult.to_dict() results
       - `metadata: dict` — strategy name, sources, version
       - `to_dict()` returning a JSON-serializable dict
       - `to_json(path: Path)` method that writes `json.dumps(self.to_dict(), indent=2, default=str)` to path

    4. **`run_signal_scan(scan_plan: ScanPlan, output_dir: Path, *, scan_date: date_cls, format: str, console: Console | None) -> ScanSignalReport`** — main orchestrator:
       - Build `CompositeTrendStrategy` with `EnhancedSuperTrendStrategy()` and `MTESv3TrendStrategy()` as sources (use lazy import from `agent.src.strategies.trend` as shown in the context section above)
       - For each `SymbolPlan` in `scan_plan.symbols`:
         - Primary timeframe = `symbol_plan.timeframes[0]` (typically "1d" — per research Pitfall 1)
         - Load parquet via `read_local_parquet(symbol_plan.cache_paths[0])`
         - If read fails or df is None/empty: append `SymbolSignalResult(symbol=..., bucket="skipped", bucket_reason=error_string, trading_signal=None, error=None)` to results
         - Run `composite.analyze(df)` -> `TradingSignal`
         - Call `classify_trading_signal(signal)` -> `(bucket, bucket_reason)`
         - Append `SymbolSignalResult(symbol=..., bucket=bucket, bucket_reason=bucket_reason, trading_signal=signal.to_dict(), error=None)` to results
         - If any Exception is raised during the above: catch `(Exception,)` only (per research Pitfall 3 — let BaseException propagate), append to `failed` bucket with `bucket="failed"` and `error=f"{type(exc).__name__}: {exc}"`
       - Build `ScanSignalReport` from collected results
       - Call `report.to_json(output_dir / "scan_results.json")`
       - If format == "table" and console is not None: call `format_signal_table(report, console)`
       - Return `report`

    5. **`format_signal_table(report: ScanSignalReport, console: Console) -> None`**:
       - Use `rich.table.Table` with title "Signal Scan Results — {scan_date}"
       - Columns: Symbol, Name, Market, Bucket, Reason, Score, Confidence
       - For each item in `report.buckets["actionable"] + report.buckets["watch"] + report.buckets["risk_excluded"] + report.buckets["skipped"]`:
         - Color bucket cell: actionable=green, watch=yellow, risk_excluded=red, skipped=dim
         - Format score and confidence from `trading_signal` fields
       - Print summary line: "N actionable | N watch | N risk | N skipped | N failed"
       - For failed items: print a second table with Symbol, Error columns
