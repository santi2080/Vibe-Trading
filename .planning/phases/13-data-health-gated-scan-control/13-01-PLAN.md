# Phase 13-01 Plan: Data Health Gated Scan Control

**Phase:** 13 | **Plan:** 01 | **Goal:** Integrate `check_watchlist_data()` as mandatory gate before `--run` execution  
**Depends on:** Phase 12 | **Requirements:** GATE-01, GATE-02  
**Created:** 2026-06-09

---

## Context

Phase 12 built the daily scan CLI foundation: `scan.py` with `--watchlist`, `--data-dir`, `--output`, `--now`, `--format`, `--run`. In plan mode (default), the CLI validates the watchlist and shows a normalized scan plan. In run mode (`--run`), it currently shows a stub message.

Phase 13 needs to wire `check_watchlist_data()` into `--run` mode so:
1. The data-health gate runs **before** any strategy analysis
2. `FAIL` blocks strategy work and produces a blocked report
3. `WARN` continues with visible caveats
4. The gate result is serialized into `data_health.json`

The existing `watchlist_data_health.py` already has:
- `check_watchlist_data(watchlist_path, data_dir, now)` → `WatchlistDataHealthReport`
- `WatchlistDataHealthReport.gate_status` → "PASS" | "WARN" | "FAIL"
- `WatchlistDataHealthReport.can_backtest` → bool
- `WatchlistDataHealthReport.to_dict()` → serializable dict
- `format_report_table(report)` → human-readable table

---

## Success Criteria

1. Every `--run` invocation executes `check_watchlist_data()` before any strategy analysis
2. `data_health.json` is written to the run directory on every `--run` execution
3. Gate `FAIL` → scan aborts, no strategy candidates, blocked `manifest.json` + `report.md`
4. Gate `WARN` → scan continues, caveats exposed in `report.md` and `manifest.json`
5. Gate `PASS` → scan proceeds normally
6. `--format json` shows gate status in the output; `--format table` shows gate table
7. `--run --dry-run` still validates but skips both the gate and strategy work
8. All existing Phase 12 tests continue to pass

### Task 13-01-1: Add gate check to scan command (15 min)

**File:** `src/cli/commands/scan.py`

Add an internal `_run_data_gate()` helper and integrate it into the `scan()` command's run-mode path.

```python
# src/cli/commands/scan.py — additions

import pandas as pd  # already needed for scan_validators

from ..data.watchlist_data_health import (
    check_watchlist_data,
    format_report_table,
)
from ..data.scan_validators import ValidationResult

def _run_data_gate(
    watchlist_path: str,
    data_dir: str,
    output_dir: str,
    now: datetime,
    dry_run: bool = False,
) -> tuple[WatchlistDataHealthReport | None, Path | None, bool]:
    """Run data-health gate. Returns (report, json_path, should_abort)."""
    if dry_run:
        return None, None, False  # --dry-run skips gate

    report = check_watchlist_data(watchlist_path, data_dir, now)
    json_path = Path(output_dir) / "data_health.json"
    json_path.parent.mkdir(parents=True, exist_ok=True)
    import json
    with open(json_path, "w") as fh:
        json.dump(report.to_dict(), fh, indent=2, default=str)

    should_abort = not report.can_backtest
    return report, json_path, should_abort
```

**Integration into `scan()` run-mode path:**
```python

report, health_json_path, should_abort = _run_data_gate(
    watchlist_path=resolved_path,
    data_dir=args.data_dir,

    now=now,
    dry_run=args.dry_run,
)

if args.format == "json":
    # Include gate status in output
    output["gate"] = report.to_dict()["gate"] if report else {"status": "skipped"}

if should_abort:
    click.echo(f"Data health check FAILED — blocking scan. See {health_json_path}", err=True)
    raise SystemExit(1)  # GATE-01: FAIL blocks strategy analysis

    click.echo(f"Data health check WARNING — {report.warnings} caveat(s). See {health_json_path}", err=True)
    # Continue — GATE-02: WARN can continue
```

```python
def test_gate_fail_blocks_scan(runner, mock_resolved_watchlist, tmp_path):
    """FAIL gate status exits 1 and writes blocked manifest."""
    # Mock check_watchlist_data to return FAIL
    with patch("agent.src.cli.commands.scan.check_watchlist_data") as mock_gate:
        mock_gate.return_value = Mock(blocking_failures=1, can_backtest=False,
                                      warnings=0, to_dict=lambda: {"gate": {"status": "FAIL"}})
        result = runner.invoke(scan, [
            "-w", str(mock_resolved_watchlist),
            "--data-dir", "data", "--output", str(tmp_path),
            "--run"
        ])
    assert result.exit_code == 1
    assert "FAILED" in result.output
    assert (tmp_path / "data_health.json").exists()
```
